# UI Rules

## DESIGN.md Priority

`DESIGN.md` is the authoritative UI design system for this project.

When `DESIGN.md` specifies a design token, component style, or layout rule, it takes priority over generic UI recommendations below.

Read `DESIGN.md` before making any UI decisions.

Design:
- mobile-first
- clean
- simple
- trustworthy
- student-friendly
- card-based

Use:
- Tailwind CSS
- shadcn/ui

Core components:
- BookCard
- RequestCard
- NoteCard
- ClassroomCard
- ImageUploader
- SchoolSelector
- GradeSelector
- TrackSelector
- PriceBadge
- ConditionBadge
- FlashcardViewer
- MessageBubble

Avoid:
- excessive animations
- dark patterns
- complicated dashboards
- admin-heavy workflows

Main navigation:
- mobile-first bottom navigation
- desktop can use sidebar or top navigation
